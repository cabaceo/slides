Intro

What?

  • Shiny allows you to rapidly build web apps using R.
  • Shinyapps.io allows you to easily deploy these apps.

Who?

  • Biostatisticians
  • Statisticians
  • Data scientists
  • R programmers

Why?

  • Improve productivity. (Shiny allows you and your team to re-run calculation procedures by clicking buttons instead of running R scripts. It's easier to share an URL than a bunch of R files.)
  • Let non-technical people use your models without knowing R.
  • Tell your data stories to the world. (Shiny supports powerful visualizations. Most shiny apps you make are mobile friendly by default.)
  • Build minimum viable products.
  • New career as Shiny developers.
  • Not hard & fun.

Demo

Simple Examples

  1. Interactive plots. For example, a bar chart that shows the actual values when you mouse over the bars. A timeseries plot that changes as you select different periods.

  2. Turn your ideas into reality. For example, this app allows you to look up a user's most recent readings on Douban (豆瓣). Notice you can also embed google ads in the app. This app provides a cleaner interface to search restaurants on Tabelog in English.

  3. Embed data analysis in your blog post so that readers can play with the results. Example.

Complex Examples

  1. Super easy to run calculations under different scenarios or statistical procedures with different input parameters. For example, calculate sample sizes in multi-regional clinical trials.

  2. Train and backtest various models, compare results and generate reports. For example, here's an app that does all of that for survival models.

  3. You can build an entire product/SaaS platform using Shiny. Example.

  4. More examples.

What again?

Shiny = statistical (R) programming + web technologies.

It allows statisticians who don't know web technologies to easily make and publish interactive web apps.

Install R packages

pkgs = c("shiny", "shinydashboard"
         # , "plotly"
         )
for (pkg in pkgs) {
        if (!pkg %in% installed.packages()) {
                cat(paste(pkg, "missing, will attempt to install\n"))
                install.packages(pkg)
        } else cat(paste(pkg, "installed OK\n"))
}

Hello, World!

library(shiny)

# make UI ---
ui <- fluidPage(
        titlePanel("Hello World!")
)

# make server ---
server <- function(input, output) {}

# make Shiny app ----
shinyApp(ui = ui, server = server)

Inside UI

Input & output elements:

  • textInput()
  • sliderInput() # run ?sliderInput to learn how to use it
  • plotOutput()

By and large, put output elements inside of mainPanel(), and input elements inside of sidebarPanel().

What is Reactivity?

The Server keeps monitoring the UI. Whenever there is a change in the UI, the Server will follow some instructions (run some R code) accordingly and update the UI's display.
